Form Validation

It is highly suggested to use AngularJS to manage form validation. However, form validation can also be done manually through your own means. You should still follow the Foundation guidelines for laying out the correct HTML elements and using the specific CSS classes.

There are two main states for validation, success and error. Success is when the user's input is valid. Error is used when the input is invalid. The classes has-success and has-error. These classes are applied to the element which has the form-group class.

Angular Forms

The desire is to move to Angular forms. By using the ng-form directive developers can gain many powerful features for validation and manipulation of user input.

The example below is created using Angular forms. While this is a simple example, it should give a good overview of how to use basic form validation in Angular. For a complete guide to using AngularJS Forms read the Developer Guide - Forms page.

To create an Angular form you can use the ng-form directive. Avoid using form element because in the HTML standard there should only be one form element per page. When declaring the directive you should give your form a name.

Within the form create your label and input groups using the form-group class on div elements. With required fields you should add the required class to the form-group and add required as an attribute to the element used for user input.

By providing a name for the form on the ng-form directive you can access it in the current scope. By also using the name attribute on the user input element you can do the same. This allows you to get the state of a single control within the form or the state of the form as a whole.

For example, this allows you to show help text for a required field that is incorrectly filled out. The following example will not show the help text until the user enters data and the data is invalid.

Using the state of the form is useful when you want to disable buttons based on the validity of the user's input. You can use the ng-class directive to set a disabled class on the button as well as set the ng-disabled directive to prevent the form from being submitted.
