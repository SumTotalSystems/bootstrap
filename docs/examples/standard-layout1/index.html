The rest of this page gives an explanation on how to build this default page.

If you view source on this page you'll see more than you actually need. The page is using AngularJS components which manipulate the DOM. Aside from that there are a couple of items that should be pointed out about the page.

  • The first three meta tags relate to page and viewport setup. The second two are optional.
  • The order of the css and JavaScript includes are important. Vendor code is included before Foundation and Foundation is included before any page or pillar css or JavaScript.
  • Foundation uses AngularJS so it is important that the app be declared on the body element using the format as shown in the example.
  • Notice that the page's content is placed within the sumt-leftnav directive.
  • The breadcrumb at the top of the page is created using an ol element with a breadcrumb class applied.

The left navigation is driven with a string for the user's name and a JSON object for the navigation sections. The HTML above shows the sumt-leftnav directive within the generalPageController controller. The controller has two properties, user and panels which are used to drive the header and panels attributes on the sumt-leftnav directive.

The property user is an object which contains information about the user. When declaring the sumt-leftnav the value user.name is placed into the directive's header attribute to set the header on the left nav. If a static header is desired a string literal can be placed into the directive's attribute: header="'My Awesome Header'"

The panels attribute of the sumt-leftnav directive takes a JSON object aray to build the navigation sections . Look at the panels property in the below JavaScript example to see its structure.

Here are the properties on the controller's scope:

{% highlight js %} $scope.user = { username: 'myUsername', name: 'John Smith' }; $scope.panels = [ { "title": "Heading1", "id": "Heading1", "children": [ { "title": "subHeading1", "id": "subHeading1", "children": [] }, { "title": "subHeading2", "id": "subHeading2", "children": [] } ] }, { "title": "Heading2", "id": "Heading2", "children": [ { "title": "subHeading3", "id": "subHeading3", "children": [] }, { "title": "subHeading4", "id": "subHeading4", "children": [] } ] }, { "title": "Heading3", "id": "Heading3", "children": [ { "title": "subHeading5", "id": "subHeading5", "children": [] }, { "title": "subHeading6", "id": "subHeading6", "children": [] } ] }, { "title": "Heading4", "id": "Heading4", "children": [ { "title": "subHeading7", "id": "subHeading7", "children": [] }, { "title": "subHeading8", "id": "subHeading8", "children": [] } ] } ]; {% endhighlight %}

If there is a footer it should be placed into the content area of the sumt-leftnav. If buttons are to be used the button element is used. Add the btn class to all button elements. For the primary, right most button, the class btn-primary must also be added. All other buttons will have btn-default sumt-btn-default applied in addition to the btn class.

Text in the footer is right aligned. In order for buttons to appear in the correct order please note how they are ordered in the example.
