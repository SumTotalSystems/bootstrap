Cards

When you don't want your DOM in a box you can put your DOM in a card. Postage not needed!

Cards display detailed data in a flexible grid layout. Employee and activity information make good candidates for display in a card layout.

To create a card layout place a container with sumt-cardContainer onto your page. You can then add any number of elements with sumt-card class.

Not representative of pillar components

The examples below show how content can appear in a card. You may notice that some of this content looks similar to pillar components. These examples should not be taken as a redline or how such a component should look like in the actual pillar. The foundation styles give a baseline for the card container and cards. The content within the cards is minimally styled so that each pillar may apply the necessary styles for its own unique set of information.

Basic Example

Any content may be placed within multiple sumt-card-content elements. Certain elements pick up special styling which is unique to items within a card. The p element will loose its margin. The header tags will also have margin removed and applied depending on where they are in the card. For img elements you may use the Bootstrap utility class pull-left to make image appear on the left side of the card. To make a footer area add a button with an additional class of pull-right inside of the sumt-card-content element for right alignment.

Horizontal Example

This example shows the basic the structure of a card which could appear in the organizational chart. Note the use of multiple sumt-card-content elements. These elements break areas into sections within the card. The "header"e;, or first section, uses the Bootstrap pull-left on the img element to pull it to the left side of the card. This allows the content to flow around the image. The user's name, inclosed in a h3 element, is specially styled within the card. Also note that the i element has had a right margin applied so that text will not run up against it.

Vertical Example

A less common use of the card is to display a vertical list of data. When it is necessary to do this apply the card class as sumt-card card-vertical to have the card inherit the special vertical styling. The vertical card applies special styling to ul elements. All li elements in the list will gain a border. If a h3 comes before the list an additional darker border is set on the top of the list. Also note the image in this vertical card is using the Bootstrap center-block utility class to center the image within the card.
