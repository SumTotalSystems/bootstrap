Contributing

SumTotal Foundation Bootstrap was never intended to be developed in a closed fashion. The idea is that Foundation becomes a toolbox for all SumTotal developers to use while building our products. There are such a vast amount of use cases for each of our product pillars that we simply cannot anticiapte every control or style required. To overcome this we want all developers at SumTotal to contribute to this project.

We imagine that a scenario for contributing code goes something like this: You're developing a new page and notice that there is a similar control (or styling) that UX has defined. You then create a reusable control (or style) that can be applied in the many places you would normally have to provide markup. At this point you pull the latest Foundation code, merge your source into the codebase, provide unit tests, and submit a pull request.

At this point the internal Foundation group will receive the request to pull in your new control (or style). We'll review the code for quality and confirm with UX that it should be reusable across all product pillars. Once it pases this threshold the code will be merged into Foundation and can be used by all developers.

Development EnvironmentInstallation Instructions

As previously mentioned Foundation Bootstrap uses NodeJS to build and maintain structure of the code. You'll need to install several prerequisites before you can start working with the source code. To begin install the following:

  • NodeJS
  • Git (for Windows)

You may want to read this blog post regarding the setup of TortoiseGit and MysysGit in a Windows environment.

  1. Clone the repository: git clone http://team2:8080/tfs/DefaultCollection/Foundation/_git/foundation-controls using either TortoiseGit or from a command prompt.
  2. Open up a NodeJS command prompt as an Administrator
  3. Install global node module dependencies: npm install -g gulp bower grunt-cli mocha jshint karma-cli
  4. Go into the foundation-controls directory and install all NodeJS dependencies: npm install
  5. Install the bower components: bower install
  6. Ready to run!
    • Build a distributable package: gulp (outputs to dist/ folder)
    • JavaScript code analysis: gulp jshint
    • Serve example pages: gulp serve
    • Run unit tests: gulp test or gulp karma
